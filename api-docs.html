Backend API Documentation

This document details the public external API structure required to integrate the frontend with the Node/Express backend.

1. General Status Endpoints

GET/api/status

Test route used to verify if the frontend successfully connected to the backend.

Expected Response (200 OK)

{
  "status": "ok",
  "message": "Backend is connected to Frontend!"
}

2. Authentication Endpoints

POST/api/auth/register

Registers a new Student or Faculty member. Admin registration is restricted.

Request Payload (Student)

{
  "name": "Jane Doe",
  "email": "jane@example.com",
  "password": "securepassword123",
  "role": "student",
  "department": "Computer Science",
  "rollNumber": "CS2026001"
}

Request Payload (Faculty)

{
  "name": "Dr. John Smith",
  "email": "john.smith@example.com",
  "password": "securepassword123",
  "role": "faculty",
  "department": "Computer Science"
}

Expected Responses

201 Created

{
  "message": "Registration successful",
  "token": "eyJhbGciOiJIUzI1NiIsIn...",
  "user": {
    "id": "60b...123",
    "name": "Jane Doe",
    "email": "jane@example.com",
    "role": "student"
  }
}

400 Bad Request if the role is admin, or duplicate email/rollNumber found.

POST/api/auth/login

Authenticates users (Admin, Faculty, and Student) using email and password, returning a JWT token.

Request Payload

{
  "email": "jane@example.com",
  "password": "securepassword123",
  "role": "student" // Can be "admin", "faculty", or "student"
}

Expected Responses

200 OK

{
  "token": "eyJhbGciOiJIUzI1NiIsIn...",
  "user": {
    "id": "60b...123",
    "name": "Jane Doe",
    "email": "jane@example.com",
    "role": "student"
  }
}

400 Bad Request if invalid credentials or role.

404 Not Found if user file not found.
